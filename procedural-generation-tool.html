Procedural Island Generation Tool

Project Github Repository: Github

Summary of Project

This project was created as part of my dissertation during my MComp. The aim was to create a tool for entirely procedurally generated islands to attempt to test the limits of PCG techniques to imitate a island created by game artists.

Feature List

Name Description
Triplanar Mapping Used to correctly texture steeper generated surfaces such as mountain sides
Custom water shader Created with the help of Catlike Coding tutorials to improve the visual appeal of the project
Height-based texturing Texturing is controlled by Texture Data layer presets which can be created and switched out for different biomes
Basic vegetation system Done by reusing the layers system from the texturing, instead using percentage values for spawning done per vertice of a chunk
LOD switching system The islands have a chunk based LOD system with auto generated LOD models of each island, done by skipping vertices in mesh generation
Auto Updating functionality There is an option to turn on auto-update so you can see the impact of parameters in real time
Perlin Noise generator Basis of the entire project which controls how the islands are generated and has a lot of fine tuning options

Highlights of the Tool and how it works

On playing the project a large grid of islands will be procedurally generated, the only parts not generated is the textures themselves and the models for the vegetation. In terms of the islands themselves, the meshes of each island have 3 auto-generated LOD models done by skipping vertices within the mesh on generation. These meshes are then textured, with the help of a custom height-based texture shader using a layer system. Then, trees are placed through the same layer system off of spawning percentages per vertice on the island to form the final island.

To finish the aesthetic side a water shader is added to make it more visually appealing. Then there is a chunk loading system which spawns island chunks based on defined view distances. The LOD model of each island is reduced based off distance from the player position, represented by a red cube in the sky. This system results in high detail close to the player and at distance, the general silhouette of each island remains the same but reducing the vertice count dramatically increasing performance.

The project was created with fine tuning and customisablity so the UI was integrated into the inspector window in the Unity Editor, this is also done through scriptable data objects for texturing, terrain and noise control which can be dragged and dropped to apply. As a note, this project was heavily researched as part of my dissertation and had a list of sources used can be provided upon request.

Improvements to the Tool

Here's a basic summary of the main points that I think could be improved and these are discussed in more detail below.

  • Expand on existing scriptable objects to allow for generation of different island biomes
  • Overhaul of the vegetation system as it tanks the FPS however, this is mainly due to a lack of LOD models for the vegetation
  • Build upon the existing 32 bit island generation prototype to allow for larger island chunks
  • Add the capability to export and import island mesh files but also allow for importing of existing heightmaps into the tool

So, previously I mentioned the usage of scriptable data objects and the idea of drag and dropping these different texture objects into the editor. The expansion would basically be getting the procedural generation, to utilise this by determining island biomes and applying the corresponding scriptable data objects. The values which dictate how biomes are decided would be a fine tunable value within the UI allow for generation is desert islands, possibly volcanic zones.

The main issue with the vegetation is the lack of any sort of LOD switching due to only having 1 LOD variant, and using the same approach as the islands would result in disfigured trees so that isn't possible. This would most likely be built into the existing LOD island generation but as a slight fix to help I could disable the generation of the leaves from the trees. This is only if the tree models allow that which would reduce the vertice could but also, flicking of trees in the distance.

Within the 16 bit recommended version of the project there is a strict max size of islands due to an issue with Unity meshes by default only allowing for 65k vertices, currently the islands are 241x241. When switching to the 32 bit version we can now support 1000x1000 size meshes however, the fps was sub 30 mainly due to the 28k vegetation objects at full LOD which clearly scales very poorly. The reason I created the vegetation generator that way knowing the issues it poses to performance, is simply due to wanting to see how the islands look visually when populated with vegetation as a prototype. The main improvements is linking the generator to the existing LOD system, however, rather than enabling LOD for just per island chunk it must be done also by distance from the player with billboarding etc.

This tool currently generates islands based off a seed so it is deterministic meaning the same islands can be recreated. However, there is no way to nicely import or export a data file to automatically apply the same attributes to recreate a previous mesh etc. The solution to this to use the existing scriptable objects but be able to save all current settings that being all data objects into a file, which can be used to recreate the same scriptable objects.
