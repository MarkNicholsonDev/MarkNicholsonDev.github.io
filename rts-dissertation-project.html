RTS Pathfinding with Formation Flocking

Project Github Repository: Github

Summary of Project

This project was created as part of my 4th year dissertation during my MComp and the time for development was a little over 2 weeks. The aim was to create a system to control units of soldiers and pathfind the unit efficently around a map and specifically through tight spaces such as a gate, minimising disruption to the unit formation.

Bare in mind this project is still in progress and some changes may be made and content added etc.

Feature List

Name Description
Unit Selection system Allows for box select, shift select to select multiple units and individual selection
Formation Flocking algorithm The main focus of the project. This is effectively a modified flocking algorithm overwriting the boid cohesion rule and using vector summation to control the direction the soliders move in to stay in formation.
Unit Controller & Pathfinder Each unit is controlled by a empty GameObject Unit which uses a Unity pathfinding agent to path the entire unit around the map and provides unit-wide control such as controlling all aspects of the formation such as the width and depth of the unit and any unit orders and then passes them to the boids to execute them.

Highlights of the Sytem and how it works

The Unit system is rather complex and contains a few main parts those being the formation management, the boid/soldier behaviour and pathfinding(Gate Detection).

A brief summary is that each boid has Cohesion and Seperation behaviours which control where they move to, the boid cohesion moves the soldier to their desired position in the formation relative to the unit controller by using a formation offset value stored on the soldier. The unit when created, calculates these formation offsets based off the desired unit width which can be defined in the inspector but also whenever the unit width is changed at any time. The unit has a Unity pathfinding agent attached which moves the unit around the terrain meaning a unit of 120 soldiers is moved by a single pathfinding agent.

For Unit selection, there are 3 main ways of selecting: Box Select, Shift left clicking(Or box selecting) and just left clicking on a soldier in the unit. The most complex is box selection so that works by taking the starting left click position on screen and the position where is is released, from that we determine 4 corners which we then raycast from. Then a box collider is created between the 4 world positions raycast to, then any unit contained within is gives each soldier under it's control a selection component which turns them red. If the ground is clicked at any point the selection is released.

As a note, this project was heavily researched as part of my dissertation and had a list of sources used can be provided upon request.

Improvements to the Tool

Here's a basic summary of the main points that I think could be improved and these are discussed in more detail below.

  • Expand on existing boid behaviour to have Alignment for rotation control of the unit.
  • Add local object avoidance boid behaviour using RVO2 or ORCA.
  • Impove the move orders to take into account the overall orientation of the unit and the click drag to set unit width.
  • Reorganisation of the unit if the formation changes, if the soldier should take the closest position to limit soldiers moving across the entire formation which doesn't make too much sense.
