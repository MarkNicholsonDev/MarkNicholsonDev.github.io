Multiplayer FPS Prototype Team Project

Project Github Repository: Github

Summary of Project

This project was created as part of my MComp. The aim was to create a splatoon-like multiplayer FPS game with objectives to capture in a small arena map. In terms of my contribution, I created the Painting system for the game where we could paint onto any surface within the Level and these paints would provide varying effects to the players and enemies within the arena similarly to Portal 2 style gels.

Feature List

Name Description
Painting of objects Primary game mechanic in the game used to help maneuver around the map with speed and jump paints along with other utility paints, e.g. blinding, slow and slippy paints to hinder enemies.
Objective flags Randomly placed around the map to be collected to win the game.
Multiplayer and solo and splitscreen Allows for many ways to play the game, we tested with 8 players in 4v4 and the server supported it well.
PS4 edition of the game A custom PS4 renderer was developed alongside, however due to Sony SDK rules I can't release the code for that section.

Highlights of my contribution and how it works

The painting works by storing a paint mask with each paintable game object which is a blank texture with the same aspect ratio as the object's texture. This paint mask is what is painted to and then combined with the original texture in the main scene shader to be applied over top of the original texture to create the appearance of painted surfaces.

The mask is painted to by doing a raycast from the crosshair position to the world and then a compute shader is ran on all objects within the paint radius to ensure all objects nearby are painted to create accurate splatting. The compute shader workspace is for every 3 indices(triangles) within the mesh and first we check if the triangle is facing the direction being painted from to reduce paint leaking through to the other side of an object which doesn't face the splat via a scalar projection between the direction to the ray origin and the normal of the triangle.

If the triangle faces the ray origin we do a sphere triangle checkto see if the current vertice is within the range of the paint. If it is, then we calculate the corresponding UV in the object texture and paint that same UV in the paint mask resulting in accurate painting of surfaces which face in the direction of the splat/paint position.

Improvements to the Painting system

One of the key issues with the system at this time is that it only works on objects without duplicated UV coordinates which can be quite awkward if using assets on the asset store. To solve this you can generate new UVs for the object in Blender which is a pain, if the idea is used in a studio with artists this is less of an issue and more of a consideration.

In terms of performance it's hard to gauge since the prototype lacks any other demanding graphics but the frame rate stayed way above 250fps in demonstrations.
